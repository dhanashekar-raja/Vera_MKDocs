# Secure Collaboration (Vera) Version 3.25.0
#### *February 2026*

## Enhancements
- **Vera SaaS**
    - Tomcat has been upgraded from the Rocky/RHEL-packaged Tomcat 9.0.87 to 9.0.106, reducing exposure to known Tomcat vulnerabilities while keeping the connector deployment and upgrade experience consistent across environments.

- **Windows Client**
    - Secure Collaboration (Vera) extends its existing AutoCAD integration to support AutoCAD 2024.
    - Secure Collaboration (Vera) extends its existing Nitro PDF support to deliver full functionality with Nitro PDF 14 on Windows, ensuring complete compatibility across all secure collaboration workflows.

- **Mac Client**
    - Secure Collaboration (Vera) adds support for saving changes to files protected by Advanced Access Control on macOS, allowing users to directly save updates to secured documents.

## **Fixes**
- **Vera SaaS**
    - Fixed an issue where Vera connector servers were flagged during vulnerability scans. The connector and server logging stack were updated to replace Apache Log4j 1.x with Log4j 2 and to update SLF4J/Logback dependencies, remediating the identified Log4j 1.x vulnerabilities.
    - Fixed an issue where Google Chrome intermittently failed to detect the installed Vera desktop application. When opening a secured document, affected users were repeatedly prompted to download and install Vera even though the application was already installed and running.
    
    !!! info "**IMPORTANT:** For more information, refer to the *Chrome and Edge Local Network Access Policy Configuration Guide* on the Secure Collaboration (Vera) Admin Portal."

- **Windows Client**
    - Fixed an issue in Adobe Acrobat where user can insert or replace pages from a secured PDF into an unsecured or secured file. This operation is now blocked, and users see a message indicating that the action is not allowed on secured files.
